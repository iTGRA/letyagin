### Announcement-плашка (над хедером, опционально)

**Структура:** одна строка текста + стрелка-ссылка, справа крестик на закрытие.

**Правила тона:**
- Одно предложение, не больше 90 символов
- Заканчивается стрелкой `→` или точкой
- Без восклицательных знаков и каплока
- Без «Только сегодня!» и «Последний шанс!»

**Примеры формулировок:**
```
Зимнее предложение — две ночи от 12 000 ₽ в историческом центре Самары →

Новогодняя ночь в ЛетягинЪ-ХОЛЛ — свободные места до 15 декабря →

Скидка 15% при бронировании от 3 ночей — промокод VESNA26 →

Банкет на День матери — специальное меню от «Дуси», до 20 ноября →
```

**Кнопка закрытия:** крестик `×` в правом углу. После закрытия не показывается до следующей сессии браузера.

## 3. Формы — общие правила

### 3.1 Обязательные и необязательные поля

- **Обязательные** помечены звёздочкой `*` после названия поля
- **Необязательные** подписаны словом «(необязательно)» после названия
- **Плейсхолдеры** подсказывают, а не повторяют название

### 3.2 Формат телефона

**Маска ввода:** `+7 (___) ___-__-__`
**Placeholder:** `+7 (___) ___-__-__`
**Валидация:** 11 цифр, начинается с +7
**Сообщение об ошибке:** «Телефон должен содержать 11 цифр, начинающихся с +7»

### 3.3 Формат email

**Placeholder:** зависит от контекста (смотри конкретную форму)
**Валидация:** стандартная email-валидация
**Сообщение об ошибке:** «Проверьте формат email — похоже, есть опечатка»

### 3.4 Согласие на обработку персональных данных

**Единый текст чекбокса на всех формах:**
```
Согласен на обработку персональных данных. [Политика обработки →]
```

Ссылка «Политика обработки →» открывает отдельную страницу `/policy` в новой вкладке.

**Валидация:** чекбокс обязательный, без галки кнопка submit неактивна (disabled).

### 3.5 Кнопки submit

**В состоянии покоя:** текст соответствует действию («Отправить заявку», «Отправить», и т.д.)
**При клике:** `[Название действия]...` — троеточие в конце (пример: «Отправляю...»)
**При ошибке:** возврат в обычное состояние, сообщение об ошибке над формой

## 10. Состояния элементов на сайте

### 10.1 Плашки «нет данных»

Когда в админке пусто и нет данных для показа (например, нет текущих акций, нет сезонного меню, нет отзывов).

**Формулировка — в стиле бренда, не «лог пустой»:**

- **Нет активных акций:** не показываем плашку вообще
- **Нет сезонного меню:** плитка сезонного меню не отображается в блоке 5 `/restaurant`
- **Нет отзывов:** блок целиком не показывается (технически маловероятно, но закладываем)
- **Нет мест в блоке «Лучшее рядом»:** блок не отображается в Phase 2

### 10.2 Загрузка изображений

Пока фото грузится — показываем **мягкую заглушку в цвете `Snow Drizzle`** (CSS-фон). Без «крутящихся спиннеров» — спиннеры дешевят бутик-сайт.

### 10.3 Скелетоны для карточек

Пока данные грузятся:
- Серые плитки в цвете `Snow Drizzle` вместо фото
- Тонкие серые полоски вместо текста
- Без анимации «shimmer» — спокойный статичный скелетон

### 10.4 Кнопки в состоянии «disabled»

Когда кнопка недоступна (например, submit формы без галочки согласия):
- Цвет — `Rum Raisin` с прозрачностью 30%
- Курсор — `not-allowed`
- Без hover-эффекта

## 11. Мобильная специфика

### 11.1 «Тап на телефон» vs «копирование»

**Десктоп:** телефон `+7 987 979-00-00` подчёркнут как ссылка. По клику — открывается диалог «скопировать номер».

**Мобильная:** тот же номер. По клику — **сразу инициируется звонок** через `tel:+79879790000`.

### 11.2 Fixed bottom bar (опционально для мобильной)

Снизу экрана — липкая полоса с быстрыми кнопками:

```
[📞 Позвонить]          [📅 Забронировать]
```

Показывается на всех страницах кроме `/contacts` (там уже есть эти CTA).

### 11.3 Тап-зоны

Минимальная высота кликабельной зоны на мобильной — **44 px** (стандарт iOS / Android). Это касается всех кнопок, ссылок в меню, элементов форм.

## 14. Архитектурные заметки для Orchid

Всё управляемое через админку — в одном справочнике **системных текстов**:

```
SiteTexts (singleton — справочник всех микротекстов)
  - navigation_labels (JSON)           — ссылки в меню и их названия
  - cta_buttons (JSON)                 — все формулировки кнопок
  - announcement_bar_enabled (bool)    — показывать ли верхнюю плашку
  - announcement_bar_text (string)     — её текст
  - announcement_bar_link (string)     — её ссылка (опционально)
  - cookie_notice_text (text)          — текст cookie-уведомления
  - 404_title (string)                 — заголовок 404
  - 404_description (text)             — текст 404
  - 500_title (string)                 — заголовок 500
  - 500_description (text)             — текст 500
  - widget_loading_text (string)       — «Загружаем виджет бронирования...»
  - widget_failed_text (string)        — «Не удалось загрузить форму...»

FormTemplate (справочник форм)
  - form_id (string)                   — «corporate», «events», «contacts», «callback»
  - title (string)
  - subtitle (string)
  - fields (JSON array)                — конфигурация полей
  - submit_text (string)                — «Отправить заявку»
  - submit_loading_text (string)        — «Отправляю...»
  - success_message (text)
  - error_message (text)
  - consent_text (text)                 — текст чекбокса согласия
  - consent_link_url (string)           — ссылка в тексте согласия
```

**Зачем:** отель может сам менять формулировки через админку без участия разработчика. Особенно важно для:
- Плашки акций (включается/выключается ежемесячно)
- Текста «Отправляющей» кнопки (если захочется поменять на «Одну секунду…»)
- Сообщений успеха (если захочется добавить «получите письмо на почту» после внедрения email-рассылки)
